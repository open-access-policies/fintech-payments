# Customer Onboarding Due Diligence Procedure (FIN-PROC-001)

### 1. Purpose

To define the step-by-step process for conducting Customer Identification Program (CIP) and Customer Due Diligence (CDD) during new customer onboarding, ensuring compliance with BSA requirements and FIN-POL-002.

### 2. Scope

This procedure applies to:

- All new customer account openings
- Individual and legal entity customers
- All products requiring customer accounts
- All onboarding channels (digital, in-person)

### 3. Overview

This procedure establishes the workflow for customer identification, verification, risk assessment, and beneficial ownership collection during onboarding per the CDD Final Rule and USA PATRIOT Act Section 326.

### 4. Procedure

#### 4.1 Individual Customer Onboarding

| **Step** | **Who** | **What** |
|----------|---------|----------|
| **1** | Customer / System | Customer provides required CIP information: |
| | | - Full legal name |
| | | - Date of birth |
| | | - Residential address |
| | | - SSN or government ID number |
| **2** | Onboarding System | Captures government-issued identification document (driver's license, passport, state ID). |
| **3** | Onboarding System / Analyst | Verifies identity through: |
| | | - Documentary verification (ID document authenticity) |
| | | - Non-documentary verification (credit bureau, public records) |
| **4** | Screening System | Screens customer against: |
| | | - OFAC SDN list |
| | | - Internal negative lists |
| | | - Other required watchlists |
| **5** | Onboarding Analyst | Reviews screening results and dispositions any matches per FIN-POL-003. |
| **6** | Risk Engine / Analyst | Assigns initial risk rating based on risk factors (geography, occupation, products). |
| **7** | Onboarding System | If verification successful and no sanctions match, proceeds to account activation. |
| **8** | Onboarding Analyst | If high-risk, triggers enhanced due diligence workflow (Section 4.3). |

#### 4.2 Legal Entity Customer Onboarding

| **Step** | **Who** | **What** |
|----------|---------|----------|
| **1** | Customer / System | Customer provides entity information: |
| | | - Legal name |
| | | - Principal place of business address |
| | | - EIN or equivalent identification number |
| | | - State/country of formation |
| **2** | Customer | Completes Beneficial Ownership Certification Form identifying: |
| | | - All individuals owning 25%+ of the entity |
| | | - One individual with significant control responsibility |
| **3** | Onboarding System | Collects for each beneficial owner: |
| | | - Full legal name |
| | | - Date of birth |
| | | - Residential address |
| | | - SSN or passport number |
| **4** | Onboarding System / Analyst | Verifies entity through: |
| | | - Formation documents (articles, certificate of formation) |
| | | - Secretary of state records |
| | | - Business information databases |
| **5** | Onboarding System / Analyst | Verifies beneficial owners using risk-based procedures similar to individual CIP. |
| **6** | Screening System | Screens entity AND all beneficial owners against: |
| | | - OFAC SDN list |
| | | - Other required watchlists |
| **7** | Onboarding Analyst | Reviews screening results and dispositions any matches. |
| **8** | Risk Engine / Analyst | Assigns initial risk rating considering entity type, industry, geography, and ownership. |
| **9** | Onboarding System | If verification successful and no sanctions match, proceeds to account activation. |
| **10** | Onboarding Analyst | If high-risk, triggers enhanced due diligence workflow. |

#### 4.3 Enhanced Due Diligence

| **Step** | **Who** | **What** |
|----------|---------|----------|
| **1** | Onboarding Analyst | Identifies EDD triggers: |
| | | - High-risk country |
| | | - PEP status |
| | | - High-risk business type |
| | | - Complex ownership |
| | | - Adverse media |
| **2** | Onboarding Analyst | Gathers additional information: |
| | | - Source of funds/wealth |
| | | - Purpose of account |
| | | - Expected activity |
| | | - Business references |
| **3** | Onboarding Analyst | Conducts enhanced screening: |
| | | - Adverse media search |
| | | - PEP database check |
| | | - Additional sanctions lists |
| **4** | Onboarding Analyst | Documents EDD findings and recommendation. |
| **5** | BSA/AML Officer or Designee | Reviews EDD case and makes approval decision. |
| **6** | Onboarding Analyst | Documents decision and proceeds with activation or rejection. |

### 5. Standards Compliance

| **Procedure Step(s)** | **Standard/Framework** | **Control Reference** |
|-----------------------|------------------------|-----------------------|
| **4.1, 4.2** | USA PATRIOT Act | Section 326 |
| **4.1, 4.2** | CIP Regulations | 31 CFR 1020.220 |
| **4.2** | CDD Final Rule | 31 CFR 1010.230 |
| **4.1-4.3** | OFAC Regulations | 31 CFR Part 501 |
| **4.3** | BSA Regulations | 31 CFR Chapter X |

### 6. Artifact(s)

- Customer application and CIP information
- Identity verification documentation
- Beneficial Ownership Certification Form (legal entities)
- Sanctions screening results
- Risk rating documentation
- EDD documentation (if applicable)
- Approval records

### 7. Definitions

| **Term** | **Definition** |
|----------|----------------|
| **CIP (Customer Identification Program)** | The program for verifying customer identity per USA PATRIOT Act Section 326. |
| **Beneficial Owner** | An individual owning 25%+ of a legal entity or with significant control responsibility. |
| **Documentary Verification** | Verification using documents such as government-issued ID. |
| **Non-Documentary Verification** | Verification using information from other sources like credit bureaus. |

### 8. Responsibilities

| **Role** | **Responsibility** |
|----------|-------------------|
| **Customer** | Provide accurate and complete information. Submit required documentation. |
| **Onboarding System** | Capture information. Route for verification. Apply automated checks. |
| **Onboarding Analyst** | Review verification results. Disposition screening matches. Conduct EDD. |
| **BSA/AML Officer** | Approve high-risk customers. Provide guidance on difficult cases. |
