# Privacy Notice Delivery Procedure (PRV-PROC-001)

### 1. Purpose

To define the process for creating, delivering, and maintaining privacy notices to customers as required by GLBA Regulation P and PRV-POL-001.

### 2. Scope

This procedure applies to:

- Initial privacy notices to new customers
- Annual privacy notices to existing customers
- Revised privacy notices
- Privacy notice content management

### 3. Overview

This procedure establishes the workflow for privacy notice preparation and delivery per GLBA Regulation P requirements, ensuring customers receive required disclosures about information collection, use, and sharing practices.

### 4. Procedure

#### 4.1 Initial Privacy Notice Delivery

| **Step** | **Who** | **What** |
|----------|---------|----------|
| **1** | Customer Onboarding | During account opening, initiates privacy notice delivery. |
| **2** | System / Customer Service | Delivers initial privacy notice before or at establishment of customer relationship: |
| | | - Online: Display during application process |
| | | - In-person: Provide paper copy |
| | | - Mail: Include with account welcome materials |
| **3** | System | Documents notice delivery date and method. |
| **4** | System | Captures customer opt-out election if provided. |
| **5** | Customer Onboarding | Files delivery confirmation in customer record. |

#### 4.2 Annual Privacy Notice Delivery

| **Step** | **Who** | **What** |
|----------|---------|----------|
| **1** | Privacy Team | Annually reviews eligibility for delivery exception: |
| | | - Only disclose NPI under exceptions |
| | | - No changes to privacy practices |
| **2** | Privacy Team | If exception applies, documents basis for not delivering annual notice. |
| **3** | Operations Team | If exception does not apply, prepares annual notice delivery: |
| | | - Generate customer mailing list |
| | | - Prepare notice materials |
| | | - Schedule delivery |
| **4** | Operations Team | Delivers annual notice by approved method: |
| | | - Mail with account statement |
| | | - Separate mailing |
| | | - Electronic delivery (with consent) |
| **5** | Operations Team | Documents delivery completion. |

#### 4.3 Revised Privacy Notice

| **Step** | **Who** | **What** |
|----------|---------|----------|
| **1** | Privacy Team | Identifies need for revised notice due to: |
| | | - New sharing practices |
| | | - New categories of NPI collected |
| | | - New third-party sharing |
| | | - Other material changes |
| **2** | Privacy Team | Prepares revised privacy notice with updated disclosures. |
| **3** | Legal / Compliance | Reviews revised notice for regulatory compliance. |
| **4** | Privacy Team | Determines if opt-out opportunity required. |
| **5** | Operations Team | If opt-out required: |
| | | - Delivers revised notice with opt-out mechanism |
| | | - Allows 30+ days for customers to opt out |
| | | - Processes opt-out responses |
| **6** | Operations Team | After opt-out period, implements new sharing practices. |
| **7** | Privacy Team | Updates master privacy notice. |

#### 4.4 Privacy Notice Content Management

| **Step** | **Who** | **What** |
|----------|---------|----------|
| **1** | Privacy Team | Maintains master privacy notice including: |
| | | - Categories of NPI collected |
| | | - Categories of NPI disclosed |
| | | - Categories of third parties |
| | | - Opt-out rights explanation |
| | | - Security description |
| **2** | Legal / Compliance | Reviews privacy notice annually for accuracy. |
| **3** | Privacy Team | Updates notice for regulatory changes. |
| **4** | Privacy Team | Maintains version history of privacy notices. |
| **5** | Privacy Team | Ensures consistent notices across all channels. |

#### 4.5 Opt-Out Processing

| **Step** | **Who** | **What** |
|----------|---------|----------|
| **1** | Customer Service / System | Receives customer opt-out request via: |
| | | - Toll-free number |
| | | - Written request |
| | | - Online form |
| **2** | Customer Service | Verifies customer identity. |
| **3** | System | Records opt-out election in customer profile. |
| **4** | System | Applies opt-out to relevant sharing processes. |
| **5** | Customer Service | Confirms opt-out to customer within **[Number, e.g., 30]** days. |

### 6. Artifact(s)

- Current privacy notice (all versions)
- Delivery confirmation records
- Exception documentation (annual notice)
- Opt-out election records
- Version history

### 7. Definitions

| **Term** | **Definition** |
|----------|----------------|
| **Privacy Notice** | A clear and conspicuous disclosure of privacy policies and practices. |
| **Delivery Exception** | Exemption from annual notice requirement when specific conditions are met. |
| **Opt-Out** | A customer's election to prevent certain information sharing. |

### 8. Responsibilities

| **Role** | **Responsibility** |
|----------|-------------------|
| **Privacy Team** | Maintain privacy notice content. Manage notice updates. Document exceptions. |
| **Legal / Compliance** | Review notice content. Ensure regulatory compliance. Approve changes. |
| **Operations Team** | Execute notice deliveries. Document delivery completion. |
| **Customer Service** | Process opt-out requests. Handle customer inquiries. |
