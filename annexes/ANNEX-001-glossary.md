# ANNEX-001: Glossary of Terms

## Purpose

This annex provides definitions for key terms used throughout the **[Company Name]** ISMS policy set, with particular focus on fintech, payment security, and regulatory terminology.

---

## Regulatory and Compliance Terms

| **Term** | **Definition** |
|----------|----------------|
| **BSA (Bank Secrecy Act)** | U.S. federal law requiring financial institutions to maintain AML programs and file certain reports to detect and prevent money laundering. |
| **CDD (Customer Due Diligence)** | The process of identifying and verifying customers, understanding customer relationships, and conducting ongoing monitoring. |
| **CIP (Customer Identification Program)** | Program required by USA PATRIOT Act Section 326 for verifying customer identity when opening accounts. |
| **CTR (Currency Transaction Report)** | Report filed with FinCEN for currency transactions over $10,000. |
| **EDD (Enhanced Due Diligence)** | Additional scrutiny and information gathering required for high-risk customers. |
| **FinCEN** | Financial Crimes Enforcement Network - U.S. Treasury bureau administering BSA. |
| **GLBA (Gramm-Leach-Bliley Act)** | Federal law requiring financial institutions to protect consumer financial information and provide privacy notices. |
| **KYC (Know Your Customer)** | Overall process of understanding customer identity, business purpose, and risk level. |
| **NPI (Nonpublic Personal Information)** | Personally identifiable financial information as defined under GLBA. |
| **OFAC** | Office of Foreign Assets Control - U.S. Treasury agency administering economic sanctions. |
| **PCI DSS** | Payment Card Industry Data Security Standard - security requirements for entities handling payment cards. |
| **Regulation E** | Federal Reserve regulation implementing the Electronic Fund Transfer Act, providing consumer protections. |
| **Regulation P** | CFPB regulation implementing GLBA privacy requirements. |
| **SAR (Suspicious Activity Report)** | Report filed with FinCEN when suspicious activity potentially involving money laundering is detected. |
| **SDN (Specially Designated National)** | Individual or entity designated by OFAC whose property must be blocked. |

## Payment Security Terms

| **Term** | **Definition** |
|----------|----------------|
| **AOC (Attestation of Compliance)** | Form attesting to PCI DSS assessment results. |
| **ASV (Approved Scanning Vendor)** | Organization approved by PCI SSC to conduct external vulnerability scanning. |
| **CDE (Cardholder Data Environment)** | People, processes, and technologies that store, process, or transmit CHD or SAD. |
| **CHD (Cardholder Data)** | At minimum, the full PAN. May include cardholder name, expiration date, and service code. |
| **CVV/CVC/CAV** | Card Verification Value/Code - security codes on payment cards. |
| **EMV** | Global standard for chip-based payment cards (Europay, Mastercard, Visa). |
| **P2PE (Point-to-Point Encryption)** | Standard encrypting card data from POI to secure decryption environment. |
| **PAN (Primary Account Number)** | Unique payment card number identifying issuer and cardholder account. |
| **PCI SSC** | Payment Card Industry Security Standards Council - organization managing PCI standards. |
| **PED (PIN Entry Device)** | Device used for PIN entry in card transactions. |
| **PFI (PCI Forensic Investigator)** | Qualified investigator approved by PCI SSC for breach investigations. |
| **POI (Point of Interaction)** | Initial point where card data is read from a payment instrument. |
| **POS (Point of Sale)** | Location and system where retail transactions are completed. |
| **PTS (PIN Transaction Security)** | PCI SSC program for POI device security validation. |
| **QSA (Qualified Security Assessor)** | Company approved by PCI SSC to conduct PCI DSS assessments. |
| **ROC (Report on Compliance)** | Detailed report documenting PCI DSS assessment by a QSA. |
| **SAD (Sensitive Authentication Data)** | Full track data, card verification codes, and PINs/PIN blocks. |
| **SAQ (Self-Assessment Questionnaire)** | Validation tool for self-evaluating PCI DSS compliance. |
| **Tokenization** | Process of replacing sensitive data with non-sensitive tokens. |
| **TPSP (Third-Party Service Provider)** | Entity providing services involving CHD on behalf of another entity. |

## Information Security Terms

| **Term** | **Definition** |
|----------|----------------|
| **AES** | Advanced Encryption Standard - symmetric encryption algorithm. |
| **APT (Advanced Persistent Threat)** | Sophisticated, targeted cyberattack campaign. |
| **CVSS** | Common Vulnerability Scoring System - standard for assessing vulnerability severity. |
| **DLP (Data Loss Prevention)** | Technology and processes to prevent unauthorized data transmission. |
| **HSM (Hardware Security Module)** | Dedicated hardware for managing cryptographic keys. |
| **IaC (Infrastructure as Code)** | Managing infrastructure through code rather than manual processes. |
| **IAM (Identity and Access Management)** | Framework for managing user identities and access rights. |
| **IDS/IPS** | Intrusion Detection/Prevention System. |
| **MFA (Multi-Factor Authentication)** | Authentication requiring multiple verification factors. |
| **RBAC (Role-Based Access Control)** | Access control based on user roles. |
| **SAST (Static Application Security Testing)** | Analyzing source code for vulnerabilities without executing. |
| **DAST (Dynamic Application Security Testing)** | Testing running applications for vulnerabilities. |
| **SIEM** | Security Information and Event Management system. |
| **SOC (Security Operations Center)** | Team monitoring and responding to security events. |
| **TLS** | Transport Layer Security - cryptographic protocol for secure communications. |
| **WAF (Web Application Firewall)** | Firewall protecting web applications. |

## Financial Operations Terms

| **Term** | **Definition** |
|----------|----------------|
| **ACH** | Automated Clearing House - electronic network for financial transactions. |
| **AML (Anti-Money Laundering)** | Laws, regulations, and procedures to prevent money laundering. |
| **Beneficial Owner** | Individual owning 25%+ of legal entity or with significant control. |
| **BIN (Bank Identification Number)** | First 6-8 digits of a payment card identifying the issuer. |
| **Correspondent Bank** | Bank providing services on behalf of another bank. |
| **FATF** | Financial Action Task Force - international AML/CFT standard-setter. |
| **MTL (Money Transmitter License)** | State license required for money transmission activities. |
| **MSB (Money Services Business)** | Business providing money services (transmission, exchange, etc.). |
| **PEP (Politically Exposed Person)** | Senior political figure, family members, and close associates. |
| **Structuring** | Breaking up transactions to avoid reporting thresholds. |
| **SWIFT** | Global messaging network for international wire transfers. |
| **Travel Rule** | Requirement to transmit originator/beneficiary information with transfers. |
