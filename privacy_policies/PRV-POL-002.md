# NPI Protection Policy (PRV-POL-002)

## 1. Objective

This policy establishes **[Company Name]**'s requirements for protecting the security and confidentiality of customer nonpublic personal information (NPI) in compliance with the GLBA Safeguards Rule (16 CFR Part 314). It defines the administrative, technical, and physical safeguards required to protect customer information from unauthorized access, use, or disclosure.

## 2. Scope

This policy applies to:

- All nonpublic personal information (NPI) as defined by GLBA
- All systems that store, process, or transmit NPI
- All employees, contractors, and service providers with access to NPI
- All locations where NPI is accessed or stored
- The entire lifecycle of NPI from collection to disposal

## 3. Policy

**[Company Name]** shall implement and maintain a comprehensive information security program to protect the confidentiality, integrity, and availability of customer NPI in accordance with the GLBA Safeguards Rule.

### 3.1 Information Security Program

A comprehensive information security program shall protect customer NPI per GLBA § 314.4.

**Program Elements:**
- Risk assessment identifying reasonably foreseeable risks
- Safeguards to control identified risks
- Regular testing and monitoring of safeguards
- Oversight of service provider arrangements
- Evaluation and adjustment of the program

**Program Documentation:**
- Written information security program
- Policies and procedures documented
- Program approved by the Board

### 3.2 Qualified Individual

A qualified individual shall be designated to oversee the information security program per GLBA § 314.4(a).

**Designation:**
- The **[Role Title, e.g., Chief Information Security Officer]** is designated as the Qualified Individual
- Designation is documented and approved by the Board
- Qualified Individual has appropriate authority and resources

**Responsibilities:**
- Oversee and implement the information security program
- Report to the Board on program status
- Coordinate with other business areas
- Stay current on security threats and practices

**Reporting:**
- Written report to the Board at least annually
- Report covers program status, risk assessment, control testing, and incidents
- Additional reporting for material security events

### 3.3 Risk Assessment

Risk assessments shall identify and evaluate risks to NPI per GLBA § 314.4(b).

**Risk Assessment Scope:**
- Risks in each area of operation that handles NPI
- Reasonably foreseeable internal and external threats
- Likelihood and potential damage of threats
- Sufficiency of existing safeguards

**Risk Assessment Process:**
- Conducted at least annually
- Updated when significant changes occur
- Documented with methodology and results
- Results used to inform safeguard selection

**Risk Categories:**
- Employee risks (negligence, malicious insider)
- Technology risks (system failures, cyberattacks)
- Physical risks (facility security, device theft)
- Service provider risks
- Other operational risks

### 3.4 Safeguard Design and Implementation

Safeguards shall be designed and implemented to control identified risks per GLBA § 314.4(c).

**Access Controls (§ 314.4(c)(1)):**
- Limit access to NPI to authorized personnel
- Implement least privilege principles
- Require authentication for NPI access
- Review access permissions periodically

**Physical Security (§ 314.4(c)(2)):**
- Protect premises containing NPI
- Control physical access to NPI storage areas
- Secure equipment containing NPI
- Protect NPI during transport

**Data Inventory (§ 314.4(c)(3)):**
- Identify and classify all NPI
- Document NPI storage locations
- Track NPI throughout its lifecycle
- Maintain data flow documentation

**Encryption (§ 314.4(c)(3)):**
- Encrypt NPI in transit over external networks
- Encrypt NPI at rest where feasible
- Use strong encryption standards
- Manage encryption keys securely

**Secure Development (§ 314.4(c)(4)):**
- Implement secure development practices
- Evaluate security of in-house developed applications
- Test applications for vulnerabilities
- Address identified security issues

**Multi-Factor Authentication (§ 314.4(c)(5)):**
- Implement MFA for accessing NPI systems
- MFA required for remote access
- Approve equivalent controls where MFA not feasible
- Document any MFA exceptions

**Secure Disposal (§ 314.4(c)(6)):**
- Securely dispose of NPI no longer needed
- Use methods that prevent reconstruction
- Document disposal activities
- Require disposal of service provider copies

**Change Management (§ 314.4(c)(7)):**
- Document changes to information systems
- Assess security impact of changes
- Test changes before implementation
- Maintain change logs

**Monitoring (§ 314.4(c)(8)):**
- Monitor systems for unauthorized access or use
- Log security-relevant events
- Review logs for anomalies
- Alert on suspicious activity

### 3.5 Security Testing

Regular testing shall assess safeguard effectiveness per GLBA § 314.4(d).

**Continuous Monitoring:**
- Ongoing monitoring of information systems
- Security tool alerts and responses
- Performance monitoring

**Periodic Testing:**
- Vulnerability assessments
- Penetration testing
- Control testing
- Testing frequency based on risk

**Testing Documentation:**
- Document testing methodology and scope
- Record test results
- Track remediation of findings
- Report significant findings to management

### 3.6 Security Training

Personnel shall receive security training per GLBA § 314.4(e).

**Training Requirements:**
- Security awareness training for all personnel
- Role-based training for security-sensitive roles
- Training upon hire and periodically thereafter
- Training on emerging threats

**Training Content:**
- Protecting customer information
- Recognizing security threats
- Reporting security incidents
- Following security procedures

**Training Documentation:**
- Track training completion
- Maintain training records
- Update training content regularly

### 3.7 Service Provider Oversight

Service providers with access to NPI shall be appropriately overseen per GLBA § 314.4(f).

**Service Provider Selection:**
- Due diligence before engagement
- Evaluate service provider security capabilities
- Assess risks of service provider arrangements

**Contractual Requirements:**
- Contracts require appropriate safeguards
- Contracts address confidentiality obligations
- Right to audit or assess compliance

**Ongoing Oversight:**
- Monitor service provider security
- Review service provider compliance periodically
- Address identified issues promptly

### 3.8 Incident Response

Security incidents shall be managed per GLBA § 314.4(h).

**Incident Response Plan:**
- Documented incident response procedures
- Defined roles and responsibilities
- Communication protocols
- Recovery procedures

**Incident Response Activities:**
- Detect and respond to security events
- Contain and mitigate incidents
- Investigate and document incidents
- Recover from incidents

**Notification:**
- Notify affected customers as required by law
- Report to regulators as required
- Document notification decisions

See RES-POL-001 (Incident Response and Business Continuity Policy) for detailed incident response requirements.

### 3.9 Board Reporting

The Board shall receive reports on the information security program per GLBA § 314.4(i).

**Annual Report:**
- Overall status of the information security program
- Risk assessment results
- Control testing results
- Security incidents
- Recommendations for improvements

**Report Content:**
- Material matters relating to the program
- Issues identified and management's response
- Changes to the program
- Emerging threats and trends

### 3.10 Program Maintenance

The information security program shall be updated based on changing circumstances per GLBA § 314.4(g).

**Program Updates:**
- Update based on risk assessment results
- Update based on testing results
- Update based on incident lessons learned
- Update based on business or technology changes

**Annual Review:**
- Review program effectiveness annually
- Assess need for program changes
- Document review and any changes

## 5. Definitions

| **Term** | **Definition** |
|---|---|
| **Customer Information** | Any record containing NPI about a customer, whether in paper, electronic, or other form. |
| **Information Security Program** | The administrative, technical, and physical safeguards used to protect customer information. |
| **Nonpublic Personal Information (NPI)** | Personally identifiable financial information, and any list, description, or grouping derived from such information. |
| **Qualified Individual** | The person designated to oversee and implement the information security program. |
| **Safeguards** | Controls implemented to protect customer information from threats. |
| **Service Provider** | Any entity that receives, maintains, processes, or otherwise has access to customer information through its provision of services. |

| **Compliance Team** | Monitor regulatory changes. Conduct assessments. Oversee service providers. |

| **All Employees** | Follow security policies. Protect customer information. Report security concerns. |
